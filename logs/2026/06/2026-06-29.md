# 2026-06-29 Worklog — Echo 캐릭터 이미지 생성봇

## 문제정의

디자인 작업 시 Echo 이미지를 매번 **같은 컨디션(비율·색·표정 등)으로 생성하기 어려움**이 있다. 결과물마다 캐릭터가 미묘하게 달라져 일관성이 깨지고, 그때마다 수작업으로 맞추느라 시간이 든다.

이를 해결하기 위해 **마스터 프롬프트와 이미지 생성봇**을 만들어, 누구나 자세·표정만 입력하면 일관된 Echo 이미지를 생성할 수 있게 한다.

**기대 효과**
- 캐릭터 일관성 확보 — 매번 동일한 컨디션의 Echo 생성
- 시간·리소스 절약 — 이미지마다 수작업 보정하던 공수 제거
- 타 팀 자립화 — 디자이너 손을 거치지 않고도 다른 팀이 직접 Echo 이미지 생성 → 디자인팀 리소스 분산 방지

## 목적

캐릭터의 **자세·표정만 요청하면 일관성을 유지한 채 뚝딱 생성되는 봇**을 만든다. 봇이 마스터 프롬프트와 베이스 이미지를 **이미 내장**하고 있어, 매번 캐릭터 설정을 입력할 필요가 없다.

## 1차 시도

**1. 베이스 이미지 확보**
- Echo의 정면(서있기)·달리기·응원(눈감고 웃기) 포즈 3장을 기준(Canonical Model Sheet)으로 선정.
- 검정 배경 PNG. 이 3장이 캐릭터의 "정답 외형" 역할.

**2. 마스터 프롬프트(캐릭터 바이블) 정리 — 먼저 함**
- 베이스 이미지를 토대로 아이덴티티/2등신 비율/컬러(HEX)/눈·입·귀·머리장식·턱받이·꼬리 규칙/재질·라이팅/금지사항/가변 템플릿을 문서화.
- 실제 이미지와 한 줄씩 대조하며 보정:
  - v1.1: 초안
  - v1.2: 눈(흰자 많은 → 꽉 찬 다크브라운), 입(항상 열림 → 기본 닫힌 미소), 배경 매번 명시, "글↔이미지 충돌 시 이미지 우선" 규칙 추가
  - v1.3: 비율 고정 블록 추가, 꼬리 흰색 금지 강화

**3. HTML 생성봇 제작**
- 마스터 프롬프트(기본 탑재) + 베이스 이미지 3장(git raw URL 자동 로딩)을 내장한 단일 HTML 파일.
- 사용 흐름: 변형 지시(자세·표정·배경) 입력 → 생성 → 결과 다운로드, "베이스로 사용" 버튼으로 재투입해 일관성 강화.

**4. 실제 생성은 미검증**
- 이미지 생성에는 외부 API(유료) 키가 필요 → 봇 동작 구현까지는 완료했으나 실제 생성 테스트는 못 함.

## 1-1. 임시방편 — GPT 웹에서 수동 생성

봇은 이미지 생성에 **유료 API가 필요**해서 바로 쓰기 어렵다. 그래서 **당장 결제 없이도 쓸 수 있는 임시방편**으로, ChatGPT 웹에서 수동으로 생성하는 방법을 마련했다.

**방법**
1. ChatGPT 웹(chatgpt.com)에 **베이스 이미지를 직접 첨부**
2. 미리 정리해둔 **생성 요청 템플릿**(마스터 프롬프트 + 변형 지시 + 장면 샘플)을 복붙
3. `[이번에 그릴 장면]`의 행동·표정·시점·배경만 바꿔 요청

**의의**
- 봇(자동화) 완성 전까지, 또는 결제 없이 써야 할 때 **즉시 활용 가능한 대안**.
- 마스터 프롬프트와 베이스 이미지만 있으면 도구는 무엇이든(GPT 웹 등) 일관 생성이 가능하다는 것도 확인.

**산출물**

## 막힌 점 / 다음에 참고

- **HTML 단독으로는 이미지 "생성" 불가** — 그림은 이미지 생성 모델이 그려야 하고, 그건 외부 API 호출이 필요. HTML은 API를 부르는 껍데기.
- **레퍼런스 기반 일관성 생성은 무료 API가 없음** — OpenAI gpt-image-1 유료, Gemini(Nano Banana)도 무료 등급 한도 0이라 결제 필요. (웹 UI는 무료지만 봇 자동화에는 API가 필요)
