# 2026-06-29 Worklog

## 목적

캐릭터(Echo)의 **자세와 표정만 요청하면 일관성을 유지한 채 뚝딱 생성되는 봇**을 만든다.

핵심은 봇이 **마스터 프롬프트와 베이스 이미지를 이미 내장**하고 있다는 것 —
사용자는 매번 캐릭터 설정을 입력할 필요 없이 "점프하며 웃는 정면" 같은 변형 지시만 주면 된다.

## 진행 흐름

1. **베이스 이미지 확보** — Echo의 정면/달리기/응원 포즈 3장을 기준(Canonical Model Sheet)으로 삼음.
2. **마스터 프롬프트 먼저 정리** — 베이스 이미지를 토대로 캐릭터 아이덴티티/비율/컬러/입·눈·꼬리 규칙/금지사항을 캐릭터 바이블로 문서화. 실제 이미지와 대조하며 v1.1 → v1.2 → v1.3으로 보정.
3. **HTML 봇 제작** — 마스터 프롬프트와 베이스 이미지 3장을 내장하고, 변형 지시 + 생성 버튼만 두는 단일 HTML 파일로 구현.
4. **실제 생성은 미검증** — 이미지 생성에 **API(유료) 키가 필요**해서, 봇 동작까지 만들었지만 실제 생성 테스트는 못 해봄.

## 막힌 점 / 다음에 참고

- **HTML 단독으로는 이미지 "생성" 불가** — 그림은 이미지 생성 모델이 그려야 하고, 그건 외부 API 호출이 필요. HTML은 그 API를 부르는 껍데기.
- **레퍼런스 기반 일관성 생성은 무료 API가 없음** — OpenAI gpt-image-1 유료, Gemini(Nano Banana)도 무료 등급 한도 0이라 결제 필요.
- 그래서 봇은 완성·보관 상태이고, **실제 생성은 결제를 켜야 검증 가능**.
