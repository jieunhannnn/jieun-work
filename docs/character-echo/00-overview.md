# Echo 캐릭터 이미지 생성봇 — 프로젝트 개요

> 캐릭터(Echo)의 자세·표정만 요청하면 일관성을 유지한 채 생성되는 봇/도구. 문제정의부터 결론까지 정리한 대표 문서.

## 문제정의

에코 이미지가 필요할 때마다, 마스터 프롬프트 없이 베이스 이미지만 GPT에 넣고 요청해왔는데, 프롬프트 없이 이미지만 던지다 보니 문제가 있었다.

- 이미지를 매번 **같은 컨디션(비율·색·표정 등)으로 생성하기 어려움**이 있다.
- 그에 따라 마음에 드는 이미지를 건지기까지 시간이 소요됐다.

이를 해결하기 위해 **마스터 프롬프트와 이미지 생성봇**을 만들어, 누구나 자세·표정만 입력하면 일관된 Echo 이미지를 생성할 수 있도록 한다.

## 1차 시도 — 마스터 프롬프트 + HTML 생성봇

캐릭터의 "정답 외형"을 문서로 고정하고, 그것을 내장한 생성봇을 만들기로 했다.

**방법**
1. **베이스 이미지 확보** — Echo의 정면(서있기)·달리기·응원(눈감고 웃기) 포즈 3장을 기준(Canonical Model Sheet)으로 선정. 이 3장이 캐릭터의 "정답 외형" 역할.
2. **마스터 프롬프트(캐릭터 바이블) 정리** — 베이스 이미지를 토대로 아이덴티티/2등신 비율/컬러(HEX)/눈·입·귀·머리장식·턱받이·꼬리 규칙/재질·라이팅/금지사항/가변 템플릿을 문서화. 실제 이미지와 한 줄씩 대조하며 보정:
   - v1.1: 초안
   - v1.2: 눈(흰자 많은 → 꽉 찬 다크브라운), 입(항상 열림 → 기본 닫힌 미소), 배경 매번 명시, "글↔이미지 충돌 시 이미지 우선" 규칙 추가
   - v1.3: 비율 고정 블록 추가, 꼬리 흰색 금지 강화
   - v1.4: 감정 표현용 요소(눈썹·불꽃 눈·찡그린 눈·눈물) 추가, 표정 수치 기준(입 벌어짐·블러셔·눈썹 각도 등) 추가, 꼬리 색 규칙 강화
3. **HTML 생성봇 제작** — 마스터 프롬프트(기본 탑재) + 베이스 이미지 3장(git raw URL 자동 로딩)을 내장한 단일 HTML 파일. 변형 지시(자세·표정·배경) 입력 → 생성 → 다운로드, "베이스로 사용" 버튼으로 재투입해 일관성 강화.

**결과**
- 봇 동작 구현까지는 완료했으나, 이미지 생성에 **외부 API(유료) 키가 필요**해 실제 생성 테스트는 못 함. → 당장 쓸 수 있는 임시방편이 필요해졌다.

**산출물**
- `20-echo-character-bible.md` — 마스터 프롬프트 (바이블 v1.4)
- `echo-bot.html` — 생성봇 (마스터 프롬프트 + 베이스 3장 내장)
- `assets/` — 베이스 이미지 3장 (front / run / cheer)

## 1-1. 임시방편 — 커스텀 GPT 만들기

봇이 유료 API 때문에 막혀, 첫 임시방편으로 **커스텀 GPT** 기능으로 캐릭터 생성봇을 만들어봤다.

**방법**
1. 지침(Instructions)란에 마스터 프롬프트 입력
2. 지식(Knowledge)란에 베이스 이미지 첨부

**결과 — 실패**
- 지식란에 올린 이미지는 이미지 **생성 과정에서 강한 시각 참조로 작동하지 않았다.**
- 업로드된 기준 이미지를 직접 분석·반영하기보다, **지침에 적힌 텍스트 설명에 더 많이 의존**하는 경향을 보였다.
- 그 결과 색감·얼굴 비율·3D 질감이 다르게 나왔고, "같은 Echo의 다른 포즈"가 아니라 **사실상 새로 만든 캐릭터에 가까운 결과**가 나왔다.

**인사이트**
- 이미지를 진짜 기준으로 쓰려면 지식란 업로드로는 부족하고, **생성 시점에 직접 첨부**해야 한다.

## 1-2. 임시방편 — GPT에서 수동 생성

커스텀 GPT가 이미지를 제대로 참조하지 못했으므로, 이번엔 **이미지를 생성 시점에 직접 첨부**하는 방식으로 GPT에서 수동 생성했다. 별도 이미지 생성 API 없이 쓰던 GPT를 그대로 활용하는 임시방편이다.

**방법**
1. GPT(chatgpt.com)에 **베이스 이미지를 직접 첨부**
2. 미리 정리해둔 **생성 요청 템플릿**(마스터 프롬프트 + 변형 지시 + 장면 샘플)을 복붙
3. `[이번에 그릴 장면]`의 행동·표정·시점·배경만 바꿔 요청

생성 요청 프롬프트 예시 (이미지 3장 첨부 + 아래 텍스트 복붙):

```
이 캐릭터 'Echo'의 가이드를 먼저 읽어줘:
https://github.com/jieunhannnn/jieun-work/blob/main/docs/character-echo/20-echo-character-bible.md

그리고 내가 첨부한 이미지 3장이 이 캐릭터의 절대 기준(모델시트)이야.
새 토끼를 만들지 말고, 첨부 이미지와 100% 동일한 Echo가 다른 자세/표정을
연기하는 모습을 그려줘. 글과 이미지가 충돌하면 무조건 이미지를 따라.

[이번에 그릴 장면]
행동: 두 손을 번쩍 들고 만세하며 폴짝 점프
표정: 눈 감고 입 벌려 활짝 웃음
시점: 정면
배경: 연한 파스텔 민트색 단색
```

**결과**
- 캐릭터 일관성은 **잘 유지됐다.** 이미지를 직접 첨부하니 커스텀 GPT(1-1)와 달리 같은 Echo가 안정적으로 나왔다.
- 하지만 **이 과정을 매번 사람이 손으로 해야 한다.** 새 채팅 열기 → 이미지 3장 첨부 → 마스터 프롬프트 복붙 → 그 다음에야 장면 입력. 이미지 한 장 뽑을 때마다 이 세팅을 반복해야 한다.
- 즉 **결과물은 만족스럽지만 작업 방식이 번거롭다.** "사람이 매번 똑같은 세팅을 반복하는" 이 부분이 자동화 대상 → 다음 단계(2차)로 이어진다.

**인사이트**
- git의 **텍스트 가이드(마스터 프롬프트 md)는 링크로 줘도 GPT가 읽는다.**
- 하지만 **이미지는 링크로 주면 베이스(참조 기준)로 활용하지 못한다** — 생성 시 그 이미지를 실제로 보고 따라 그리지 못함. 그래서 이미지는 **채팅창에 직접 첨부**해야 베이스로 인식된다.

**산출물**
- `21-request-prompt-template.md` — 생성 요청 템플릿 + 장면 샘플

## 2차 시도 — Image Starter (브라우저 자동화)

1차 봇은 유료 API가 필요해 바로 못 썼고, 1-2(GPT 수동)는 매번 이미지를 손으로 첨부해야 하는 번거로움이 있었다. 이를 없애기 위해, 쓰던 GPT를 그대로 활용하면서 **세팅 과정만 자동화**하는 도구를 만들었다.

**방법**
- **사용 흐름:** 더블클릭(또는 명령어) 한 번 → ① 새 GPT 채팅 열기 ② 기준 이미지 3장 자동 첨부 ③ 마스터 프롬프트 자동 입력 ④ 커서를 `Action:` 뒤에 위치. 사용자는 `회의 중 고민하는 표정` 한 줄만 입력 → 생성.
- **구현:** OpenAI 공식 기능이 아니라 **브라우저 자동화(Playwright)** — 스크립트가 실제 크롬을 조작해 사람이 하던 클릭·첨부·붙여넣기를 대신함. 로그인 세션을 저장(`user-data`)해 1회 로그인 후 재사용. `Echo 시작.command` 더블클릭으로 실행(터미널 불필요). `config.js`에서 베이스 이미지·마스터 프롬프트만 바꾸면 다른 캐릭터에도 재사용 가능.

**결과**
- 정상 동작 확인 — 한 번에 이미지 3장 + git 바이블 링크 + 마스터 프롬프트가 자동 세팅되고, 장면 한 줄만 입력하면 일관된 Echo가 생성됨. **이미지 수동 첨부 번거로움 제거.**

**한계 / 참고**
- GPT UI가 바뀌면 스크립트가 깨질 수 있어 **유지보수 필요** (브라우저 자동화는 공식 지원이 아님).
- **한 채팅에서 계속 생성하면 일관성이 흐려질 수 있음** — 대화가 길어질수록 GPT가 앞쪽 베이스 이미지·프롬프트를 약하게 기억함. 일관성이 중요하면 **장면마다 툴을 다시 실행해 새 채팅으로 리셋**하는 게 안전 (한 채팅당 2~3장 권장).
- **사람이 직접 요청하는 것보다 속도가 살짝 느림** — 매번 새 채팅을 열고 이미지 3장 첨부 + 프롬프트 입력을 자동으로 다시 하기 때문에, 세팅에 약간의 시간이 더 든다.
- 개인용 셋업 도구로는 충분. 팀 배포·상시 서비스로 키울 거면 유료 API가 더 안정적.

**산출물**
- `tools/echo-image-starter/` — Playwright 기반 자동화 툴 (`start.js`, `config.js`, `Echo 시작.command`, `base/` 이미지 3장)

**다른 팀도 활용 가능 🙌**
- 이 툴을 누구나 재현할 수 있게 **빌드 명세서(md)** 로 정리해뒀다. 본인 AI(Claude Code/Codex/Cursor 등)에 아래 링크를 주고 "이대로 만들어줘"라고만 하면 된다.
  - 명세서: `tools/echo-image-starter/BUILD-WITH-AI.md`
  - AI에게 줄 raw 링크: `https://raw.githubusercontent.com/jieunhannnn/jieun-work/main/tools/echo-image-starter/BUILD-WITH-AI.md`
- 에코 버전 그대로 재현되며(베이스 이미지·마스터 프롬프트·바이블 링크 포함), 다른 캐릭터로 쓰려면 `config.js`의 이미지·프롬프트만 바꾸면 된다.

**종료 시 주의**
- 다 쓰면 **터미널(또는 실행 창)을 그냥 닫으면 된다** → 자동화 크롬도 함께 종료됨.
- 단, **이미지 생성·다운로드가 끝난 뒤** 닫을 것 (생성 중이면 그것만 끝내고).
- 로그인 세션은 `user-data/`에 저장돼 있어 다시 실행해도 로그인은 유지됨.

## 결론

캐릭터 일관성 문제를 **도구로 해결**했다.

- Echo를 매번 같은 컨디션으로 그리기 어려웠던 문제를 → **마스터 프롬프트(캐릭터 바이블) + 베이스 이미지**로 "정답 외형"을 고정해 해결.
- 유료 API 장벽은 → **브라우저 자동화(Image Starter)** 로 우회. 쓰던 GPT를 그대로 활용해 **버튼 한 번 + 장면 한 줄**이면 일관된 Echo가 나오게 만듦.
- 결과적으로 **누구나(타 팀 포함) 디자이너 없이** Echo 이미지를 생성할 수 있고, 빌드 명세서로 **재현·확장도 가능**하게 정리.

**의의**

단발성 결과물이 아니라 **재사용 가능한 자산**(프롬프트·봇·자동화 툴·문서)을 남겼다. "프롬프트와 기준 이미지만 정의되면 도구는 무엇이든 일관 생성이 가능하다"는 것을 확인했고, Echo 외 다른 캐릭터로도 그대로 확장할 수 있다.
